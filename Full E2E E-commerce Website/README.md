## Overview
This is a **self-assigned manual testing project** for the [SauceDemo](https://www.saucedemo.com) website.  
The project demonstrates how requirements and application behavior are analyzed and converted into **structured manual test cases** following QA best practices.

---

## User Story
As a user, I want to **login, browse products, add items to the cart, and complete checkout** so that I can make online purchases successfully and securely.

---

## Scope of Testing
The following functionalities were covered in this project:

- Login page validation and error handling  
- Inventory / Product page navigation and filtering  
- Cart management (add, remove, multiple items)  
- Checkout process (information validation, order overview, finish order)  
- Session management (logout, reset app state)  

---

## Acceptance Criteria Covered
- Presence and validation of login fields  
- Error messages for invalid, empty, or mismatched credentials  
- Successful login redirects to inventory page with products displayed  
- Product details page shows photo, price, description, and add-to-cart option  
- Cart updates correctly when items are added or removed  
- Checkout requires valid first name, last name, and ZIP code  
- Cancel buttons redirect appropriately to cart or inventory page  
- Finish button completes the order and redirects to confirmation page  
- Reset app state and logout features work as expected
