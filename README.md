# Manual Testing Projects

## Overview
This repository contains a collection of **self-assigned manual QA testing projects** created for learning and portfolio purposes.  
Each project focuses on analyzing requirements and designing structured manual test cases for real-world application features.

The goal of this repository is to demonstrate **QA thinking**, not just theory — covering how user stories and acceptance criteria are translated into effective test cases.

---

## What This Repository Covers
- Manual test case design
- Requirement and user story analysis
- Functional testing
- Positive and negative test scenarios
- Validation and boundary checks
- Basic usability and security considerations

---

## Repository Structure
Each folder represents a **separate testing project**, and contains its own documentation and artifacts.


---

## Tools & Documentation
- Excel – test case documentation
- HTML – UX and email mockups
- GitHub – version control and portfolio showcase

---

## Learning Focus
- ISTQB-aligned manual testing practices
- Clear and structured test documentation
- Understanding application behavior from a tester’s perspective

---

## Notes
- All projects are **practice-based** and not related to real clients or production systems.
- No real user data or confidential information is used.
- This repository focuses on **manual testing only**.
