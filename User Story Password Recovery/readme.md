# Password Recovery – Manual Testing Project

## Overview
This is a **self-assigned manual testing project** for the **Password Recovery** feature of an eCommerce website.  
The project demonstrates how requirements are analyzed and converted into structured manual test cases following QA best practices.

---

## User Story
As a user who forgot my password, I want to recover my password via email so that I can regain access to my account.

---

## Scope of Testing
The following functionalities were covered in this project:

- Login page – **Forgot Password** link
- Password recovery page
- Email input validation
- Password recovery email delivery
- Password reset link behavior
- Reset password flow
- Redirection and success messaging

---

## Acceptance Criteria Covered
- Presence of “Forgot Password” link on login page  
- Redirection to password recovery page  
- Sending recovery email with a secure reset link  
- One-time-use password reset link  
- Link expiration after 24 hours  
- New password and confirm password validation  
- Successful reset redirects user to login page
